## Improvements from Phase 2.6

- Structured 5-validator pipeline ensures every script is checked for fact accuracy, location correctness, repetition, Hindi language ratio, and story structure
- Auto-regeneration loop retries failed scripts up to 3 times before marking as `failed`
- Mystery prompt now has 10 strict rules preventing hallucination and enforcing fact-grounding
- Every script is scored on hallucination, grounding, story, and language
- Report endpoint (`GET /topics/{id}/script/report`) provides full validation trace

## Remaining Weaknesses

- Fact validator uses keyword overlap (not semantic matching) — may miss subtle hallucinations or flag valid paraphrases
- Location validator whitelist depends on research data completeness
- Story validator is heuristic-based (marker words, sentence position)
- Engagement scoring remains rule-based (questions + exclamation marks + emotional words)
- LLM generation speed (~55s per call) limits practical auto-regeneration throughput
